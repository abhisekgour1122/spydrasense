Backend: Django + DRF · PostgreSQL · JWT Total pages: 36 Roles: 3 web + Participant (mobile)

Roles & Access — single login, role decides the menu (roles are fixed, not editable)

RoleAccessScopeWhere
Super AdminEverything — superset of all pages (platform: monitoring, security, AI config + all Admin work)All orgs (global)Web
AdminUsers, Assessments, Reports, Analytics, Notifications — within their orgOne org (org switcher if multi)Web
CoachReview only — assigned participants' sessions, feedback, progressAssigned participantsWeb
ParticipantTakes assessments — no web accessSelfMobile app

Creation hierarchy (who creates whom)

Super Admin creates Organization + invites Admin → Admin creates Coaches & Participants, assigns Coach → Participant → Coach reviews assigned participants' sessions → Participant takes assessment on mobile
Super Admin can do everything an Admin can (superset) — same pages, org-scoped when they enter an org.

One web login with RBAC — role decides which menu & pages are shown (no separate Coach login). Roles are fixed: Super Admin, Admin, Coach (+ Participant on the Mobile App) — there is no role-editor / permission-matrix page. Super Admin is a superset of Admin (same org pages, org-scoped when entered). Navigation: multi-page sections use tabs (Monitoring, Security, Assessments, Analytics, Notifications); lists drill into detail with a back button (Organizations, Users, Reports, Sessions). · Confidential — FutureFirst Solutions.
